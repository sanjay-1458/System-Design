
# System Design Repository

A curated repository focused on learning and implementing real-world system design concepts using practical examples, architecture discussions, UML modeling, API design, and object-oriented design principles.  

This repository is divided into two major sections:

- **Low Level Design (LLD)**  
  Covers object-oriented design problems, UML diagrams, API modeling, lifecycle flows, and commonly used software design patterns implemented in C++.

- **High Level Design (HLD)**  
  Covers scalable distributed systems, architecture patterns, infrastructure-level decisions, scalability tradeoffs, reliability, and system architecture concepts.

# Low Level Design

Low Level Design focuses on writing maintainable, extensible, and scalable software components using SOLID, object-oriented design principles.  
This section includes design patterns, UML diagrams, lifecycle modeling, API design, and implementation-oriented problem solving.

# High Level Design

High Level Design focuses on designing scalable distributed systems and understanding architectural tradeoffs used in production-grade applications.

Topics covered include:

- Scalability
- Load Balancing
- Distributed Systems
- Caching
- Message Queues
- Database Scaling
- CAP Theorem
- Reliability Engineering
- Microservices Architecture
- System Tradeoffs
